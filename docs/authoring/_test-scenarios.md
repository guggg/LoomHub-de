# 測試情境 / Test Scenarios（產製流程驗收用）

> **用途**：在**新的 session** 裡拿這些「未加工情境」測試 LoomHub-de 的產製流程——
> `分類決策 → 照對應 type 指引寫成合規 SKILL.md → 依 AGENTS.md 檢核`。
>
> **刻意不標答案是哪一類**——分類判斷本身就是要測的東西（看 `docs/authoring/README.md` §1 決策流程能不能導出合理答案）。下方「參考分類」摺在最後,測完再對。
>
> 每個情境模擬一位同事口頭描述「我做了一件事,想分享上來」,語氣鬆散、資訊不全——正是真實貢獻的起點。
>
> **這是測試用檔案,`_` 前綴表示非正式資產;正式使用前可刪。**

---

## 情境 A

> 「我最近每次接到資料異常的告警,都要先跑一串固定的 SQL 去 warehouse 撈:先看那張表最近的載入時間、再比對上游來源的筆數、再看有沒有重複 key。我把這串查詢跟判讀順序整理成一段給 AI 的指示,以後想直接叫它幫我做這個初步排查,不想每次自己貼。」

**你要做的**：判斷 type → 照指引寫成 `skills/<name>/SKILL.md` → 跑檢核。

---

## 情境 B

> 「我常常要把一段落落長的英文技術文件濃縮成中文摘要給主管看,格式固定:三句話總結、關鍵風險、建議動作。我調了一個蠻好用的 prompt 範本,裡面有幾個地方要填(文件內容、受眾、想強調什麼)。想分享給團隊,大家自己複製去填就好。」

**你要做的**：同上。

---

## 情境 C

> 「我把我們內部那台 Athena（放 log 的）包成一個可以讓 agent 直接查的服務——設定好連線、限制成唯讀、只開放 query 跟 list-tables 兩個操作。裝好之後 agent 就多了『查 log』的能力,不用我手動貼 SQL 結果。連線設定跟金鑰要另外給。」

**你要做的**：同上。特別注意 secret 要怎麼處理（見 mcp-server 指引的安全段落）。

---

## 情境 D

> 「新人進來要建一個新的 dbt model,其實有固定的步驟:先開 staging model、加來源測試、再建 intermediate、最後 mart 層加文件跟 schema test。每一步有前後順序、也有前置條件（要先有原始表、設好 profiles）。我想把這整套流程寫下來讓大家照著跑,目前沒有可執行腳本,就是步驟說明。」

**你要做的**：同上。注意末節該用「安裝」還是「取用/套用」（見 workflow 指引的決策樹）。

---

## 新 session 建議的測試指令

在新 session（已 `cd` 到本專案）可以這樣起手,逐一跑：

```
讀 docs/authoring/README.md 與 docs/authoring/_test-scenarios.md。
針對情境 A：依 README §1 決策流程判斷它該是哪個 type，說明理由；
然後照對應的 docs/authoring/<type>-guide.md 幫我起草一份合規的
skills/<name>/SKILL.md；最後依 AGENTS.md 的檢核清單自我檢核並回報結果。
```

（B / C / D 同樣方式。或若已裝 Loom,直接把情境丟給 Loom 看它怎麼判斷 + 起草。）

驗收重點：
- 分類判斷合理嗎？模糊時有沒有依 §2 原則處理（而非卡住）？
- 產出的 SKILL.md 8 欄位齊、正文小節符合該 type（§3.2.2）、demo 用約定區塊、末節取用方式正確？
- AGENTS.md 檢核有沒有真的抓出不合規？

<details>
<summary>參考分類（測完再看）/ Reference answers</summary>

| 情境 | 建議 type | 為什麼 |
|---|---|---|
| A | **`skill`** | 要 agent「自動幫我做初步排查」、不想每次手動貼 → 裝進 agent 的常駐能力。單一能力,非多步編排。 |
| B | **`prompt`** | 一段人**手動複製、填空、貼上**的固定範本 → 取用型,末節「複製 / 取用」。有變數 → 變數/參數表 + 範例輸出。 |
| C | **`mcp-server`** | 把服務包成「可掛載、給 agent 一批工具」的伺服器 → 提供的工具/資源 + 設定 Config（金鑰用 env placeholder,別寫死）+ 安裝/啟動。 |
| D | **`workflow`** | 多步驟、有前後依賴與前置條件的流程。目前無可執行腳本（純步驟）→ 末節用「取用 / 套用」（複製流程），非安裝。 |

注意 A 和 B 是最能考驗分類的一對：**同樣是「一段給 AI 的指示」,差別只在「要 agent 自動用（skill）還是人手動貼（prompt）」**——正是 README §1 決策流程第 2 問要分辨的。

</details>
